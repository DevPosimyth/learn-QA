# How to Access and Use the Workspace in WDesignKit

## 1. Create a Workspace

### Step 1: Open the Dashboard

Navigate to the Workspace section to land on the Workspace Dashboard where you can:
- View existing workspaces
- See details for each workspace (members, templates, widgets, snippets)
- Create new workspaces

### Step 2: Click "Create Workspace"

Locate the **Create Workspace** button and click it to open the setup popup.

### Step 3: Enter Workspace Details

In the popup, provide:
- **Workspace Name** — Enter a unique name (e.g., _Client Website_, _E-commerce Project_)
- **Upload an Image** — Optionally upload an image related to your workspace

### Step 4: Create Workspace

Click the **Create Workspace** button. Your new workspace will be created and displayed on your dashboard.

---

## 2. User Roles & Permissions

Each member is assigned a role determining their access level.

| Role | Permissions |
|---|---|
| **Owner** | Full control — management, billing, members, settings |
| **Editor** | Create and modify templates, widgets, snippets; cannot manage members |
| **Subscriber** | View-only access; can receive updates; no editing rights |

### How to Assign Roles

1. Click the three-dot menu on the workspace card
2. Select **Manage Members**
3. In the **Role dropdown**, select the desired role
4. Click **Save** to apply changes immediately

---

## 3. Workspace Features

### Templates Management
- **View Templates** — Open the Templates section
- **Add Templates** — Click Add Template, fill details, and save
- **Edit/Delete Templates** — Manage existing templates as needed

### Widgets Management
- **View Widgets** — Open the Widgets section
- **Add Widgets** — Click Add Widget, configure settings, and save
- **Edit/Delete Widgets** — Manage and modify widgets

### Code Snippets Management
- **View Snippets** — Open the Snippets section
- **Add Snippets** — Click Add Snippet, enter name, paste code, and save
- **Edit/Delete Snippets** — Manage stored code snippets

### Managing Members

1. Click on **Users** from the workspace overview page
2. From the **Manage Members** section:
   - View all workspace members
   - Add new members
   - Assign or change roles
   - Remove users if necessary

---

## 4. Best Practices

- Assign roles based on responsibilities
- Limit admin access to trusted users
- Review member access regularly
- Remove inactive users
- Keep content organized with clear naming conventions
- Back up important templates, widgets, and snippets

---

## Important Note

Each workspace functions independently — templates, widgets, and snippets created in one workspace are not shared with others. You can create multiple workspaces for different clients, separate websites, or testing environments.
